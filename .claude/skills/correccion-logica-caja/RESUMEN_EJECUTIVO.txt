PROBLEMA IDENTIFICADO
─────────────────────────────────────────────────────────────────────────────

La apertura de caja se está implementando como "activar una caja" (cambiar 
estado de Inactiva a Abierta), cuando en realidad debería ser "crear una 
sesión diaria" (agregar dinero a un vendedor cada día).

SÍNTOMA PRINCIPAL:
En http://localhost:8000/finanzas/caja, el modal de apertura solo lista 
cajas con estado "Inactiva". Esto significa que una caja solo se puede 
abrir UNA VEZ.

IMPACTO:
- ❌ No se puede reutilizar la misma caja al día siguiente
- ❌ Se necesitaría crear una nueva caja cada día (incorrecto)
- ❌ No hay histórico de sesiones
- ❌ Confusión en la lógica de negocio


SOLUCIÓN PROPUESTA
─────────────────────────────────────────────────────────────────────────────

Cambiar de:
  Caja (Permanente) → Estado: Inactiva/Abierta/Cerrada

A:
  Caja (Configuración) → Activa/Inactiva
      ↓
  Sesión de Caja (Operación Diaria) → Abierta/Cerrada/Validada


BENEFICIOS
─────────────────────────────────────────────────────────────────────────────

✅ Permite múltiples aperturas del mismo vendedor
✅ Separación clara: Configuración vs Operación
✅ Histórico completo de sesiones
✅ Mejor auditoría
✅ Más flexible para cambios futuros
✅ Escalable

CONCLUSIÓN
─────────────────────────────────────────────────────────────────────────────

Esta skill propone una corrección fundamental en la lógica de apertura de 
caja. El cambio es necesario para que el sistema funcione correctamente y 
sea escalable.

La implementación requiere cambios en múltiples capas (BD, Backend, Frontend),
pero el beneficio es significativo: permitir que los vendedores abran caja 
múltiples veces sin necesidad de crear nuevas cajas.

Se recomienda implementar esta solución lo antes posible.
